## GitHub workflow

- Sempre que um commit for criado neste projeto, ele também deve ser enviado para o GitHub no mesmo fluxo de trabalho.
- O remoto principal é `origin` e a branch padrão atual é `main`, salvo instrução diferente do dono do projeto.
- Antes de commitar, conferir `git status` e incluir apenas arquivos que pertencem ao escopo pedido.
- Depois do commit, executar `git push` para publicar a branch no GitHub.
